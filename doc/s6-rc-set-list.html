The s6-rc-set-list program

s6-rc-set-list lists all the services of a set, along with the sub each service is on.

Interface 

     s6-rc-set-list [ -v verbosity ] [ -r repo ] set
  • s6-rc-set-list prints all the services in set set of repository repo, one per line, to stdout. Each line contains:
    • The name of the service
    • A slash (/) character
    • The canonical name for the sub the service is in.
  • It exits 0.

Options

-v verbosity, --verbosity=verbosity
Be more or less verbose. The default is 1, which means that error messages and warnings will be written to stderr. 0 means that only error messages will be written, and 2 or more adds informational messages.
-r repo, --repository=repo
Use the repository in repo, which must exist. Default is /var/lib/s6-rc/repository.

Exit codes

0
Success.
3
Nonexistent set.
100
Incorrect usage.
102
Invalid repository.
111
System call failed.

Notes

  • s6-rc-set-list is meant to be used with automation such as cut or util-linux's column for formatting. A slash is used to separate the service name for the sub because slashes cannot appear in service names.
